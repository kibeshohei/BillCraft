以下の手順でコミットを作成してください。

1. `git status` で変更ファイルを確認する
2. `git diff --staged` でステージされた内容を確認する（ステージが空なら `git diff` も確認する）
3. ステージが空の場合は `git add -p` を使わず、ユーザーに何をステージするか確認する
4. 変更内容を分析して日本語でコミットメッセージを生成する

## コミットメッセージのフォーマット

```
<type>: <概要>

<詳細（任意）>
```

### type の一覧

| type | 使いどころ |
|---|---|
| feat | 新機能の追加 |
| fix | バグ修正 |
| refactor | 動作を変えないリファクタリング |
| style | フォーマット・スタイルのみの変更 |
| docs | ドキュメント・コメントの変更 |
| test | テストの追加・修正 |
| chore | ビルド設定・依存関係の変更 |

### ルール

- 概要は 50 文字以内
- 詳細は必要な場合のみ記載
- `console.log` の削除忘れがあれば警告する

5. 生成したメッセージをユーザーに提示して確認を取る
6. 承認されたら `git commit -m` でコミットする
