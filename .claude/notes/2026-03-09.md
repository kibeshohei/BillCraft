## やったこと

### Phase 3: 請求書作成フォーム（続き）・バグ修正

#### 1. 請求書保存エラーのデバッグ
- catch ブロックのエラーメッセージを一時的に具体的な内容に変更して原因を特定
- 原因：`Invoice validation failed: userId: Path 'userId' is required.`

#### 2. NextAuth v5 session.user.id が undefined になる問題を修正
- `auth.ts` に callbacks を追加
  - `jwt` callback: ログイン時に `token.id = user.id` を設定
  - `session` callback: `session.user.id = token.id` に代入
- 修正後は再ログインが必要（古い JWT に id が入っていないため）

#### 3. フォームの UX 改善（エラー時に入力値が消える問題）
- `app/(dashboard)/invoices/new/page.tsx` を全面的に controlled inputs 化
- `useState` でフォーム全体の値を管理する `form` オブジェクトを作成
- `Field` コンポーネントに `value` / `onChange` props を追加
- 明細行も `LineItem` 型を定義し、name / quantity / unitPrice を state で管理
- エラー発生時も入力値が保持されるようになった
- 成功時は `redirect()` でページ遷移するためリセット不要

#### 4. PDF 日本語対応
- `@fontsource/noto-sans-jp` をインストール
- Google Fonts GitHub から Noto Sans JP の TTF ファイルをダウンロード → `public/fonts/NotoSansJP-Regular.ttf`
- `app/lib/pdf/InvoiceDocument.tsx` で `Font.register()` を使ってフォントを登録
- `fontFamily: "NotoSansJP"` を全スタイルに適用
- PDF の全ラベルを英語 → 日本語に変更（Invoice → 請求書、Bill To → 請求先 など）

## 詰まりポイント

- **MongoDBドキュメントをClient Componentに渡すとエラー**: `.lean()` で取得したMongoDBドキュメントはMongoose Document形式で、Client Componentに渡すと `Only plain objects can be passed to Client Components from Server Components` エラーになる。`JSON.parse(JSON.stringify(...))` でPlain Objectに変換の必要あり。
